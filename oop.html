Orientação a Objetos para Alienígenas

Introdução

Ao longo das últimas décadas, diferentes linguagens de programação foram criadas e evoluíram para resolver problemas específicos, por mais que tenham se tornado cada vez mais abrangentes e aplicáveis a diversas finalidades. Entre os diversos aspectos desta evolução, o que nos interessa aqui são os diferentes paradigmas que surgiram. Estes paradigmas se referem à características das linguagens de acordo com o que elas possibilitam fazer mas também à maneira de criar programas. Alguns exemplos de paradigmas são: imperativo, procedural, declarativo, funcional e orientado a objetos.

A Programação Orientada a Objetos

Este paradigma de programação (também conhecido pela sigla POO) surgiu principalmente como uma alternativa à programação estruturada, muito comum entre as principais linguagens entre os anos de 1950-60 como o Algol. Na programação estruturada, os programas eram organizados em um grande bloco de código e cada instrução era seguida uma após a outra, o que poderia fazer com que os programas ficassem muito extensos e difíceis de serem entendidos.

A POO surgiu como uma alternativa oferecendo abstrações para organizar programas criando separação de responsabilidades, facilidade de manutenção, modelos para criar objetos, métodos e muito mais. Vamos explorar em mais detalhes cada um destes aspectos.

Tudo começa com os tipos primitivos. Em Java, temos tipos como String, int, double, byte, short, long e boolean. Ao declarar um tipo, um espaço é criado na memória do computador com uma quantidade de bytes pré-definida para armazenar este tipo específico de dados. Porém não é possível criar grandes programas apenas com tipos primitivos: é necessário agrupá-los em estruturas de dados como arrays, stacks, listas ligadas, grafos ou árvores.

Estas estruturas aumentam as possibilidades de criar e transformar informações (e portanto criar softwares mais úteis e sofisticados), mas também aumentam o nível de complexidade destes programas. A POO visa, portanto, organizar esta complexidade, além de resolver outros problemas como a repetição e redundância de código. Vamos investigar cada um dos principais componentes de um programa que utiliza este paradigma.

Objetos

Chamamos de objeto, no cotidiano, qualquer coisa material ou imaterial que pode ser percebida pelos nossos sentidos e descrita através de suas características. De modo similar, em matéria de programação chamamos de objeto qualquer pedaço de código que representa uma dessas coisas, como um carro, uma casa, um estudante, uma conta bancária, uma data, etc. Eles são definidos por atributos (que descrevem as características do objeto) e métodos (as ações deste objeto). Além disso, dizemos que cada objeto possui um estado, que se refere ao conjunto de valores (dos atributos) que descrevem a sua condição em determinado momento.

A título de ilustração, vamos imaginar um objeto que descreve uma pessoa. Este objeto poderia ter os atributos de nome, idade e endereço e os métodos andar, falar, comer e dormir. O código para criar esta pessoa poderia ser algo como isso:

public Pessoa {    
  	String nome = "Jonas";
    String endereco = "Av. Angélica nº 100";
    int idade = 26;
}

Agora imagine que nós precisamos criar uma segunda pessoa. Será que precisamos criar um novo objeto Pessoa novamente passando os novos valores? Não mesmo! Graças à POO e às classes.

Classes

Uma classe é um modelo ou um esquema para a criação de objetos, definindo os atributos e métodos que irão compor o objeto. Pense em uma classe como uma forma e um objeto como um bolo. A mesma forma poderá fazer infinitos bolos e, embora eles possam ser de diferentes sabores, o formato será sempre o mesmo. Vamos criar uma classe Pessoa para poder instanciar novos objetos “pessoa”, com os atributos e métodos mencionados acima:

// Declaração de classe:
public class Pessoa {
		private String nome;
    private String endereco;
		private int idade;

// Construtor:
    public Pessoa(String nome, String endereco, int idade) {
        this.nome = nome;
        this.endereco = endereco;
        this.idade = idade;
    }
}

Muito bem! Agora, se quisermos criar uma nova pessoa, tudo que precisamos fazer é criar uma nova instância desta classe. Vamos criar um objeto que represente Alan Curtis Kay, criador da linguagem Smalltalk e considerado um dos pais da orientação a objetos:

Pessoa alan = new Pessoa("Alan Kay", "Springfield, MA", 82, 178);

Agora que já entendemos os conceitos básicos de classes e objetos, vamos nos aprofundar nos quatro pilares da Orientação a Objetos: abstração, encapsulamento, herança e polimorfismo.

Abstração

Vamos novamente partir da definição geral da palavra para depois entendê-la enquanto conceito da programação. O Dicionário Online de Português nos dá a seguinte definição: “Operação mental através da qual elementos e aspectos são isolados, somente no pensamento, sendo que (na totalidade) não existem isoladamente; resultado dessa operação”. Trazendo para uma situação cotidiana, imagine que uma criança está pedalando uma bicicleta e seu pai pede para que ela freie. Mesmo que a criança não saiba os detalhes técnicos da implementação do mecanismo de freio na bicicleta (onde ao puxar a manopla, um cabo de aço puxa a estrutura de ferradura que faz com que as borrachas de freio pressionem o aro, fazendo o movimento deste diminuir com o atrito), ela sabe que se ela apertar a manopla, a bicicleta irá parar.

A ideia é a mesma no universo da programação: esconder os detalhes de implementação, focando nas funcionalidades principais. Dessa forma podemos criar uma visão simplificada de um sistema complexo ao destacar somente as informações relevantes.

Encapsulamento

Este conceito se refere à prática de criar uma camada de proteção para algumas informações no nosso código, como os atributos de uma classe. Quando criamos a classe “Pessoa”, já utilizamos a ideia de encapsulamento, representada no código pela palavra-chave private. Em Java temos três tipos de modificadores (chamados dessa forma pois modificam a visibilidade das informações), utilizados para implementar a prática de encapsulamento:

  1. public: qualquer parte do programa têm acesso à este atributo, método ou classe;
  1. private: somente a própria classe tem acesso ao atributo, método ou subclasse;
  1. protected: A própria classe ou subclasses herdadas da classe principal têm acesso. Assim, se uma classe (mesmo que esteja fora do pacote) estende da classe com o atributo protected, ela terá acesso a ele. Ou seja, o acesso é por pacote e por herança.

Além destes modificadores, também temos estruturas especiais para obter e alterar os valores marcados como private: os getters e setters. Para obter e alterar valores da nossa classe Pessoa, portanto, seria necessário implementá-los da seguinte maneira, dentro da declaração da classe:

// Getters
    public String getNome() {
        return nome;
    }
    public String getEndereco() {
        return endereco;
    }    public int getIdade() {
        return idade;
    }
    // Setters
    public void setNome(String nome) {
        this.nome = nome;
    }    public void setEndereco(String endereco) {
        this.endereco = endereco;
    }    public void setIdade(int idade) {
        this.idade = idade;
    }
}

Agora estamos prontos para alterar o atributo idade do nosso objeto que representa Alan Kay quando ele fizer aniversário:

alan.setIdade(83);

E para retornar o valor de qualquer um dos atributos, usamos os getters:

alan.getEndereco(); // Springfield, MA

Herança

Este conceito se refere à possibilidade de criar novas classes baseadas em outras já existentes. Vamos imaginar que queremos criar uma nova classe Aluno, que tem todos os atributos e métodos de Pessoa, mas também precisa de outras propriedades adicionais, como número de registro, gênero e CPF. Ao invés de criar tudo do zero e redefinir as propriedades que já foram definidas na classe Pessoa, podemos simplesmente estender esta classe

public class Aluno extends Pessoa {
    private int ra;
    private int cpf;
    private char genero;

    public Aluno(String nome, String endereco, int idade,
int ra, int cpf, char genero) {
        super(nome, endereco, idade); // chama o construtor da classe Pessoa
        this.ra = ra;
        this.cpf = cpf;
        this.genero = genero;
    }

Utilizar herança em orientação a objetos é considerado uma boa prática pois protege o encapsulamento, facilita a reutilização de código e reduz possíveis efeitos colaterais. Além do extends e do super, também temos as palavras-chave abstract e final que se referem à herança. A primeira é utilizada para definir que uma classe ou método não pode ser instanciado, apenas herdado; a segunda é contrário: quando uma classe ou método só pode ser instanciado porém não pode ser herdado.

Polimorfismo

Este último pilar da programação orientada a objetos se refere à habilidade inerente aos objetos de tomar diferentes formas ou de ter diferentes comportamentos. Nós criamos uma classe Pessoa, porém uma pessoa pode agir como um professor em sala de aula, um paraquedista saltando de um helicóptero, um violinista em uma orquestra.

Na linguagem Java, o polimorfismo pode ser alcançado através da sobreposição ou sobrecarga de métodos. A sobreposição ocorre quando uma subclasse provê sua própria implementação de um método que já tinha sido definido na superclasse. Já a sobrecarga ocorre quando uma classe possui múltiplos métodos com o mesmo nome, porém diferentes parâmetros. Vamos ver alguns exemplos:

public class Animal {
    public void fazSom() {
        System.out.println("Algum barulho");
    }
}
public class Lobo extends Animal {
    @Override
    public void fazSom() {
        System.out.println("Auuuuuuul!");
    }
}

Aqui é possível ver a sobreposição em ação (apontada pela palavra-chave Override). A classe Lobo estende a classe Animal, porém altera um método que já havia sido definido na superclasse. O próximo exemplo demonstra como opera a sobrecarga:

public class Calculadora {
    public int soma(int a, int b) {
        return a + b;
    }
    public double soma(double a, double a) {
        return a + b;
    }
}

Como definimos dois métodos de soma nesta classe, podemos passar valores inteiros ou decimais e o programa irá selecionar o método apropriado automaticamente.

Calculadora c = new Calculadora();
int resultado = c.add(2, 3); // Output: 5
double resultado2 = c.add(2.5, 3.5); // Output: 6.0

Assim, dizemos que o método soma é “polimórfico” pois pode assumir diferentes comportamentos.

Bônus: Interfaces

Para além dos conceitos já explorados, também consideramos importante entender a ideia de Interfaces para compreender melhor o paradigma da programação orientada a objetos. Elas se referem à uma coleção de métodos abstratos que descrevem o comportamento que uma determinada classe deve ter, como um contrato a ser seguido. Porém com este conceito não vamos entrar em detalhes de implementação, apenas deixar como sugestão para a continuação dos seus estudos :)
